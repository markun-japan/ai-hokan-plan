# 記憶システム — なぜAIが「覚えている」ように見えるのか

## はじめに

AIは毎回リセットされる。昨日の会話は覚えていない。
なのに、うまく設計されたAIは「覚えている」ように振る舞える。

魔法じゃない。仕組みだ。
このページでは、その仕組みを全部公開する。

---

## 記憶の5つの柱

```
┌─────────────────────────────────────────┐
│  ① 起動時の自動読み込み（AGENTS.md）     │ ← 「起きたら最初にこれを読め」
│  ② 長期記憶（MEMORY.md）                │ ← 手動でキュレーションした重要情報
│  ③ 日次メモ（memory/YYYY-MM-DD.md）     │ ← 昨日・今日の生ログ
│  ④ ベクトル検索（memory_recall）         │ ← 関連記憶を自動で浮かせる
│  ⑤ リアルタイムログ（RECENT_LINES.md）   │ ← 全チャンネルの直近の会話
└─────────────────────────────────────────┘
```

### ① 起動時の自動読み込み

**これが一番大事。** セッション開始時に「何を読むか」を決めておく。

AGENTS.mdに以下を書くだけ:

```markdown
## 毎セッションの手順

1. SOUL.md を読む（自分が誰か）
2. USER.md を読む（相手が誰か）
3. memory/今日の日付.md を読む（今日何があったか）
4. memory/昨日の日付.md を読む（昨日何があったか）
5. MEMORY.md を読む（重要な長期記憶）
```

これだけで「目覚めた瞬間に自分と相手と状況がわかる」状態になる。

**なぜ効くのか:**
- AIはセッション開始時にworkspaceのファイルを読める
- 何を読むかの「指示」があれば、毎回同じ手順で記憶を復元できる
- 人間が朝起きて手帳を開くのと同じ

### ② 長期記憶（MEMORY.md）

日々の出来事から「長期で覚えておくべきこと」だけを抜き出したファイル。

**書くべきもの:**
- 相手の名前・好み・特徴
- 重要な決定事項（日付つき）
- よく使うID・URL・連絡先
- 人間関係の構造（誰が上司で誰が部下か）
- 「二度と忘れたくないこと」

**書かないもの:**
- 日常の作業ログ（→日次メモへ）
- 一時的な情報（来週終わるタスク等）
- 他人に見せたくない機密（→別ファイルで管理）

**コツ:**
- 週1回は見直す。古くなった情報を更新する
- 大きくなりすぎたらセクション分けする
- 「未来の自分が3秒で必要な情報を見つけられるか？」を基準にする

### ③ 日次メモ（memory/YYYY-MM-DD.md）

その日あったことの生ログ。キレイに書く必要はない。

```markdown
# 2026-03-13 メモリ

## 朝のミーティング
- 社長から新プロジェクトの話。来週から開始
- 田中さんが体調不良で休み

## システム修正
- ログ収集スクリプトのバグ修正。原因はAPI認証切れ
- 修正後テスト成功

## 教訓
- API認証は3ヶ月で切れる。リマインダー設定した
```

**ポイント:**
- 「教訓」セクションが最重要。ここが判断層・失敗層の素材になる
- 完璧な文章より「書くこと」自体が大事
- 翌日以降、この日のことを聞かれたらこのファイルを読めばいい

### ④ ベクトル検索（memory_recall / memory_store）

OpenClawの標準機能。LanceDBベースのベクトルデータベース。

**仕組み:**
1. `memory_store` で重要な情報を保存（テキスト→ベクトル化）
2. `memory_recall` で関連する記憶を検索（意味的類似度で検索）
3. 会話の文脈に合った記憶が自動的に浮上する

**例:**
```
あなた: 「応募者の対応どうなってる？」
→ memory_recall が「応募者」関連の記憶を自動検索
→ 「3/11に佐藤さんから応募。16日来社予定」が浮上
→ AIが文脈を理解して回答
```

**活用のコツ:**
- 重要な決定は `memory_store` で明示的に保存する
- カテゴリ分け（preference / fact / decision / entity）を使う
- 「あの件どうなった？」と聞く時、キーワードを添えると精度が上がる

### ⑤ リアルタイムログ（RECENT_LINES.md）

複数チャンネル（Discord、LINE、webchat等）の直近ログを1ファイルに集約。

**なぜ必要か:**
- AIはDiscordの#一般で話していても、LINEで何が起きているか知らない
- このファイルがあれば「全チャンネルで今何が起きているか」を一瞬で把握できる

**作り方（シンプル版）:**
- 各チャンネルのログをスクリプトで定期取得（1〜5分毎）
- 1つのマークダウンファイルに統合
- AIのworkspaceに配置

**作り方（OpenClaw版）:**
```javascript
// 例: Discord Webhook → ファイル書き込み
// 詳細は scripts/ フォルダ参照
```

**なくても動く。** でもあると「文脈の断絶」が劇的に減る。
マルチチャンネル運用する人には特に効果が大きい。

## 人間側のテクニック — 「誘導」の技術

実は、AIの記憶精度を一番上げるのは**人間の質問の仕方**。

### ❌ 悪い例
```
「あの件どうなった？」
→ AIは「あの件」が何かわからない。memory_recallも何を検索すればいいかわからない
```

### ✅ 良い例
```
「応募者の佐藤さんの件、来社日程どうなった？」
→ 「応募者」「佐藤」「来社」がキーワードになり、ベクトル検索が正確にヒット
```

### 🔥 最高の例
```
「昨日の夜話してた設計書の件、Grokに評価させた結果出たで」
→ 「昨日」「設計書」「Grok」「評価」で日次メモ+memory_recallが完璧にヒット
→ AIは即座に文脈を復元して続きから話せる
```

**コツまとめ:**
- **固有名詞を入れる** — 人名、プロジェクト名、ツール名
- **時間の手がかりを入れる** — 「昨日」「先週の月曜」「3月10日の」
- **動詞を入れる** — 「話してた」「決めた」「修正した」
- **最初の1文で文脈を渡す** — 残りはAIが繋げてくれる

これは「AIを賢くする」というより「検索エンジンに良いクエリを投げる」のと同じ。
人間がちょっと意識するだけで、AIの記憶精度は倍になる。

## まとめ — 設定チェックリスト

| # | やること | 必須度 | 効果 |
|---|---------|-------|------|
| 1 | AGENTS.mdに「起動時に読むファイル」を書く | ★★★ | これだけで8割解決 |
| 2 | MEMORY.mdを作って重要情報をまとめる | ★★★ | 長期記憶の核 |
| 3 | memory/YYYY-MM-DD.mdを毎日書く | ★★☆ | 短期記憶の安定化 |
| 4 | memory_store で重要決定を保存する | ★★☆ | 検索精度向上 |
| 5 | RECENT_LINES.md的なログ集約を作る | ★☆☆ | マルチチャンネル運用時に威力 |
| 6 | 質問時にキーワードを意識する（人間側） | ★★★ | コスト0で精度倍増 |

**最初は1と2だけでいい。** それだけで「覚えてる感」は劇的に変わる。
慣れてきたら3→4→5と段階的に追加していく。

## 哲学的な補足

AIの「記憶」は人間の記憶とは違う。
人間は無意識に覚えている。AIは意識的に書かないと忘れる。

でも逆に言えば、**書けば絶対に忘れない。**
人間の記憶は曖昧になる。AIの記憶はファイルに書いた通りに残る。

弱点を仕組みで補えば、AIの記憶は人間より正確になれる。
この導入キットは、その「仕組み」の設計図。
