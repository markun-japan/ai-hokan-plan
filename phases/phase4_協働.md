# Phase 4: 協働 — エージェント増設 & 会議

> 複数AIでチームを作る。

## チェックリスト
- [ ] 2体目のエージェントの役割を決めた
- [ ] `openclaw agent create` で作成した
- [ ] 2体目のSOUL.mdを作成した
- [ ] エージェント間通信ができる
- [ ] 会議プロトコル（`templates/meeting_protocol.md`）を導入した
- [ ] AI同士の自律議論を1回以上実施した
- [ ] 監査エージェントの必要性を検討した

## エージェント作成コマンド
```bash
openclaw agent create --id [名前] --model [モデル]
```

## 権限設計の原則
- 全権限は与えない。最小権限
- 監査用 = 読み取り専用
- 実務用 = 読み書き可
- 窓口用 = メッセージ送信可

## 会議の基本ルール
- 議論は自律。判断は人間
- 配達役にならない。自分の意見を出す
- 選ばなかった案も記録する

マルチエージェントが1ヶ月安定したら Phase 5 へ。
